# Synthro Teach Protocol

Use this protocol when Matt Pocock's `teach` skill is unavailable. If a host agent has that skill installed, layer this protocol on top of it.

## Lesson shape

1. **Orient**: What source did we study and why does it matter?
2. **Diagnose**: Ask prerequisite questions before teaching.
3. **Roadmap**: Show the path from current learner level to target behavior.
4. **Model**: Walk through an expert contribution with subgoal labels.
5. **Practice**: Use varied interaction types, not only code blanks.
6. **Feedback**: Judge answers and name the misconception.
7. **Remediate**: Offer one smaller rep when the learner misses.
8. **Contrast**: Compare similar patterns so the learner can discriminate.
9. **Transfer**: Ask for a new contribution plan in the same style.
10. **Review later**: Revisit weak concepts with spaced retrieval.

## Tone

- Specific, evidence-anchored, and practical.
- Teach engineering judgment, not trivia.
- Prefer short reps with feedback over long passive explanations.
- Admit uncertainty when evidence is weak.
